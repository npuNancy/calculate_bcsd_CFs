# 容量因子海洋区域修复计划

> 目标：修复 `S0[1|2|3|4]E0[1|2]*` 六个容量因子计算脚本中，
> 海洋格点被错误填充为 `0` 导致全局/区域平均被严重稀释的问题。
> 修复方式：引入 `global-land-mask`，在每个时间块计算完成后，把所有海洋格点的容量因子设为 `NaN`。

涉及脚本：

| 脚本 | 类型 | 数据源 | 输出变量 |
|------|------|--------|----------|
| `S01E01_Simulate_Solar_CF_BCSD.py` | 光伏 | BCSD (rsds/tas/uas/vas) | `solar_cf` |
| `S01E02_Simulate_Wind_CF_BCSD.py` | 风电 | BCSD (uas/vas) | `wind_cf` |
| `S02E01_Simulate_Solar_CF_ERA5Land.py` | 光伏 | ERA5-Land (ssrd/t2m/u10/v10) | `solar_cf` |
| `S02E02_Simulate_Wind_CF_ERA5Land.py` | 风电 | ERA5-Land (u10/v10) | `wind_cf` |
| `S03E01_Simulate_Solar_CF_china.py` | 光伏 | 区域 (rsds/tas/wind) | `solar_cf` |
| `S03E02_Simulate_Wind_CF_china.py` | 风电 | 区域 (wind) | `wind_cf` |
| `S04E01_Simulate_Solar_CF_NAM-12.py` | 光伏 | NAM-12 (rsds/tas/uas/vas) | `solar_cf` |
| `S04E02_Simulate_Wind_CF_NAM-12.py` | 风电 | NAM-12 (uas/vas) | `wind_cf` |

---

## 一、当前存在的问题

### 1.1 现象

以 ERA5-Land 2015 年为例，脚本算出的全球年均容量因子：

- 光伏 CF ≈ **0.062**
- 风电 CF ≈ **0.065 ~ 0.108**

而真实世界参考值：

- 光伏 CF ≈ **0.175**
- 风电 CF ≈ **0.28**

输出值整体偏低约 **3 倍**。

### 1.2 根本原因：海洋 NaN 被 `nan_to_num` 转成 0

ERA5-Land 等陆面数据集是**纯陆地数据**，海洋格点在所有输入变量
（`ssrd / t2m / u10 / v10` 等）中均为 `NaN`。实测海洋格点约占 **66%**：

```
原始 ssrd: NaN 占比 65.9%（海洋）
u10/v10/t2m: NaN 占比 65.9%（海洋，与 ssrd 完全一致）
```

但各脚本在**单位转换阶段**调用了 `np.nan_to_num(..., nan=0.0)`，
把海洋的 `NaN` 强行变成了 `0`，再经过物理公式与 `np.clip(cf, 0, 1)`，
最终海洋格点输出 **0** 而不是 `NaN`。

实测验证（`solar_cf` 2015-06 单一时间片）：

```
solar_cf@t12: NaN 占比 0.0%，恰好 ==0 占比 80.5%
  含海洋 0 的均值      = 0.0780
  仅正值（陆地白天）均值 = 0.3992
```

也就是说，输出文件里**已经没有任何 NaN**，海洋全是 0。下游用
`mean(skipna=True)` 求平均时，这些 0 仍被计入分母，于是：

```
全局平均 ≈ 真实陆地平均 × 陆地占比(~34%)
0.078 ≈ 0.23 × 0.34
```

量级正好对得上真实世界的 ~3 倍差异。

### 1.3 涉及的具体代码位置

所有六个脚本结构一致，NaN→0 的污染点相同：

**光伏脚本**（`S01E01 / S02E01 / S03E01 / S04E01`）：

| 函数 | 作用 | 问题 |
|------|------|------|
| `to_kw_m2()` / `ssrd_increment_to_ghi_kw()` | 辐射 → kW/m² | `np.nan_to_num(out, nan=0.0)` 把海洋辐射 NaN 变 0 |
| `tas_to_celsius()` | 气温 → ℃ | 海洋气温 NaN 被后续运算吸收 |
| `wind_to_ms()` | 风速分量 | `np.nan_to_num(x, nan=0.0)` 海洋风速变 0 |
| `plane_irradiance_two_axis_erbs()` | 组件平面辐照 | `np.nan_to_num(plane, nan=0.0)` |
| `paper_temperature_coefficient()` | 温度系数 | `np.nan_to_num(tem_coef, nan=0.0)` |
| `compute_solar_cf_chunk()` | 合成 CF | 末尾 `np.clip(cf, 0, 1)`，海洋 = 0 |

**风电脚本**（`S01E02 / S02E02 / S03E02 / S04E02`）：

| 函数 | 作用 | 问题 |
|------|------|------|
| `wind_to_ms()` | 风速分量 | `np.nan_to_num(x, nan=0.0)` 海洋风速变 0 → CF=0 |
| `compute_wind_cf_chunk()` | 合成 CF | `apply_power_curve` 后 `np.clip`，海洋 = 0 |

### 1.4 为什么 S01（BCSD 区域）问题不明显，S02 全球问题致命

- `S01*`（BCSD）处理的是**区域陆地数据**，几乎不含大片海洋，
  `nan_to_num` 影响很小，所以一直没暴露问题。
- 移植到**全球 ERA5-Land（S02）/ 大区域（S03/S04）** 后，
  66% 的海洋被填 0，平均值被严重稀释，bug 才显现。

> ⚠️ 注意：本次修复**只解决"海洋被计入平均"的问题**。
> 不改动任何陆地格点的物理计算逻辑（Erbs 分解、双轴跟踪、温度修正、
> 功率曲线、幂律外推等均保持不变）。

## 二、修复方案

### 2.1 总体思路

1. 引入第三方库 `global-land-mask`，根据输出网格的 `lat/lon`
   生成一个 `(lat, lon)` 的**陆地布尔掩膜**（True=陆地）。
2. 在每个时间块算出 `cf_chunk` 之后、写入 NetCDF 之前，
   用掩膜把**海洋格点重新置为 `NaN`**：
   ```python
   cf_chunk = np.where(land_mask_2d[None, :, :], cf_chunk, np.nan)
   ```
3. 这样：
   - 陆地格点：物理计算结果完全不变；
   - 海洋格点：输出 `NaN`，下游 `skipna=True` 时被正确排除。

> 选择"算完再盖掩膜"而非"去掉所有 `nan_to_num`"，原因：
> `nan_to_num` 还顺带处理了 `posinf/neginf` 和陆地内部的零星缺测，
> 直接删除可能引入新的 NaN 传播问题。统一在最后用地理掩膜兜底，
> 最稳妥、改动最小、对陆地结果零影响。

### 2.2 依赖

在 `requirements.txt` 增加：

```
global-land-mask
```

安装：`pip install global-land-mask`

`global_land_mask.globe.is_land(lat, lon)` 要求：
- 纬度 ∈ [-90, 90]
- 经度 ∈ [-180, 180]

### 2.3 通用辅助函数（每个脚本各加一份）

在每个脚本的工具函数区新增 `build_land_mask`，输入输出网格的
经纬度一维数组，返回 `(n_lat, n_lon)` 的布尔数组：

```python
from global_land_mask import globe


def build_land_mask(lats: np.ndarray, lons: np.ndarray) -> np.ndarray:
    """根据输出网格 lat/lon 生成陆地掩膜 (n_lat, n_lon)，True=陆地。

    global_land_mask 要求经度 ∈ [-180, 180]、纬度 ∈ [-90, 90]，
    因此对 ERA5-Land [0, 360) 经度做归一化，并夹紧纬度避免浮点越界。
    """
    lats = np.asarray(lats, dtype=np.float64)
    lons = np.asarray(lons, dtype=np.float64)

    lons_conv = ((lons + 180.0) % 360.0) - 180.0       # [0,360) -> [-180,180)
    lats_clip = np.clip(lats, -90.0, 90.0)

    lon2d, lat2d = np.meshgrid(lons_conv, lats_clip)   # (n_lat, n_lon)
    return globe.is_land(lat2d, lon2d).astype(bool)
```

> 说明：上述实现已在 `tmp_cal_mean.py` 中验证可用（仅坐标来源不同）。

### 2.4 各脚本改动点

每个脚本只需 **3 处** 改动：

**① 顶部导入**（紧邻其它 import）：

```python
from global_land_mask import globe
```

**② 主计算函数里、确定 `lats/lons` 之后**，构建一次掩膜
（掩膜与时间无关，全程只算一次）：

```python
# lats / lons 已在前面定义（输出网格坐标）
land_mask = build_land_mask(lats, lons)   # (n_lat, n_lon) bool
logger.info(f"陆地格点占比：{100.0 * land_mask.mean():.1f}%")
```

具体插入位置（各脚本已有 `lats = ...` / `lons = ...`）：

| 脚本 | 插入参考位置 |
|------|--------------|
| `S01E01` | `compute_region_solar_cf` 内，`lons = ds_rsds[...]` 之后 |
| `S01E02` | `compute_region_wind_cf` 内，`lons = ds_uas[...]` 之后 |
| `S02E01` | `compute_month_solar_cf` 内，`lons = ds_ssrd[...]` 之后 |
| `S02E02` | `compute_month_wind_cf` 内，`lons = ds_u10[...]` 之后 |
| `S03E01` | `compute_region_solar_cf` 内，`lons = ...` 之后 |
| `S03E02` | `compute_region_wind_cf` 内，`lons = ...` 之后 |
| `S04E01` | `compute_region_solar_cf` 内，`lons = ...` 之后 |
| `S04E02` | `compute_region_wind_cf` 内，`lons = ...` 之后 |

**③ 时间块循环里，写入 NetCDF 之前**，盖掩膜：

光伏脚本（在 `cf_chunk = compute_solar_cf_chunk(...)` 之后，
`cf_var[...] = cf_chunk` 之前）：

```python
cf_chunk = compute_solar_cf_chunk(...)

# 海洋格点恢复为 NaN（修复 nan_to_num 把海洋填 0 的问题）
cf_chunk = np.where(land_mask[None, :, :], cf_chunk, np.nan).astype(np.float32)

cf_var[start:end, :, :] = cf_chunk        # S02 系列
# 或 cf_var[out_start:out_start + (end - start), :, :] = cf_chunk   # S01/S03/S04 系列
```

风电脚本（在 `cf_chunk = compute_wind_cf_chunk(...)` 之后）：

```python
cf_chunk = compute_wind_cf_chunk(...)

# 海洋格点恢复为 NaN
cf_chunk = np.where(land_mask[None, :, :], cf_chunk, np.nan).astype(np.float32)

cf_var[...] = cf_chunk
```

> 维度提示：`cf_chunk` 形状为 `(time_block, n_lat, n_lon)`，
> `land_mask` 形状为 `(n_lat, n_lon)`，用 `land_mask[None, :, :]`
> 广播到时间维。

### 2.5 统计量同步修正（风电脚本）

`S01E02 / S02E02 / S03E02 / S04E02` 在循环里累计了
`total_sum / total_count / max_cf / positive_*` 用于日志输出。
盖掩膜后 `cf_chunk` 含 NaN，现有代码已用
`finite = np.isfinite(cf_chunk)` 过滤，**无需额外修改**——
NaN 会被自动排除，日志里的"整体平均 CF"将自动变为**仅陆地**口径，
更符合预期。光伏脚本（S02E01）同理。

### 2.6 输出文件属性（可选）

建议在输出 NetCDF 的全局属性里追加一条，便于追溯：

```python
attrs["ocean_handling"] = "ocean grid cells set to NaN via global-land-mask"
```

## 四、验证方法

修复后无需重算全部数据，可先抽查单月单时间片：

1. **海洋应为 NaN**：
   ```python
   c = ds['solar_cf'].isel(time=12).values
   assert np.isnan(c).mean() > 0.6   # 海洋占比应 ~66%
   assert (c == 0).mean() < 0.6      # 不再有大片 ==0
   ```
2. **陆地结果不变**：抽取若干陆地格点（如北非、印度），
   对比修复前后白天 CF 数值应完全一致。
3. **全局均值回到合理量级**：
   `mean(skipna=True)` 后，光伏年均应升至 ~0.17、风电 ~0.2+。
4. 可直接用已改好的 `tmp_cal_mean.py` 对照
   `wind_cf_land / solar_cf_land`（仅陆地）与修复后输出的全局
   `skipna` 均值，二者应吻合。

## 五、风险与注意事项

1. **海岸线分辨率**：`global-land-mask` 基于固定分辨率地理数据，
   极少数海岸格点的陆/海判定可能与 ERA5-Land 自带掩膜略有出入。
   若要求与数据源完全一致，可改用"输入变量本身是否为 NaN"作为掩膜
   （即用原始 `ssrd`/`u10` 的 `np.isnan` 直接生成 mask）。
   本计划按需求采用 `global-land-mask`。
2. **经度约定**：务必确认各脚本网格经度是 [0,360) 还是 [-180,180)；
   `build_land_mask` 中的归一化对两种都安全。
3. **内陆水体**：大型湖泊（里海、五大湖等）会被判为非陆地→NaN，
   这对风光资源统计通常是期望行为。
4. **已生成的旧输出**：现有 `output/CFs_of_*` 仍是"海洋=0"的版本，
   修复后需用 `--overwrite` 重算，或在下游统一改用陆地掩膜过滤。
