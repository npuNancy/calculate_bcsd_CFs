# 各国 & 全球「逐年平均容量因子」计算计划

## 1. 目标

基于已生成的 BCSD 降尺度风/光容量因子（CF）NetCDF 文件，计算两类逐年平均指标：

1. **各国逐年平均 CF**：对每个国家（BCSD 降尺度区域）+ china，按「未来年份」给出一个标量平均容量因子。
2. **全球逐年平均 CF**：对「N 个 BCSD 降尺度国家 + china」共 N+1 个区域的逐年平均值，**按各区域的陆地格点数加权**求平均，得到每年的全球平均 CF。
   - 即 **后者 = 前者按陆地格点数加权的平均**（大国格点多、权重更大）。

风电、光伏分别独立计算。

## 2. 输入

### 2.1 命令行参数

| 参数 | 说明 | 默认值 |
|---|---|---|
| `--model` | 气象模式 | `NESM3` |
| `--scenarios` | 未来情景，逗号分隔，可只给一个 | `ssp126,ssp245,ssp585` |
| `--energy` | `solar` / `wind` / `both` | `both` |
| `--cf-root` | CF 输出根目录 | `output` |
| `--output-dir` | 结果输出根目录 | `output/annual_mean_cf` |
| `--years` | 年份过滤 `Y0-Y1`，空=全部 | 空 |
| `--land-mask / --no-land-mask` | 是否用 global_land_mask 把海洋置 NaN 后再平均 | `--land-mask`（开） |
| `--overwrite` | 覆盖已有结果 csv | 关 |

### 2.2 输入文件结构（已存在）

- 各国（陆地降尺度区域）：
  ```
  {cf_root}/CFs_of_{energy}/{model}/{region}/{energy}_CF_{region}_{model}_{scenario}_*_allmonths.nc
  ```
  变量：`solar_cf` / `wind_cf`，维度 `(time, lat, lon)`，3 小时分辨率（2015–2060 共 134416 步）。

- china：
  ```
  {cf_root}/CFs_of_{energy}_china/{model}/{energy}_CF_china_{model}_{scenario}_*_allmonths.nc
  ```
  网格更大（china 约 348×442），单文件约 83 GB（解压后），需逐年切片读取，控制内存。

> 区域列表通过扫描 `CFs_of_{energy}/{model}/` 下的子目录动态获得，再追加 `china`。
> 文件名年份段可能不同（如 `2015-2060`、`2060-2060`），用 glob 匹配 `*_{scenario}_*_allmonths.nc`，多匹配时优先 `2015-2060`。

## 3. 计算方法

对每个 (energy, scenario, region)：

1. 打开 nc（lazy / 按需切片，不整体载入），变量取 `solar_cf` 或 `wind_cf`。
2. **海洋掩膜（默认开）**：用 `global_land_mask` 由该文件 lat/lon 构建陆地掩膜（经度归一化到 [-180,180]），把海洋格点置 NaN。
   - 与仓库现有 `S01E0x` / `utils/cal_mean.py` 一致。
   - 注意：当前磁盘上的 NESM3 文件**尚未**掩膜海洋（旧文件），脚本内自行掩膜可保证结果不依赖输入是否已掩膜。
3. **逐年平均**：先按时间求年内均值（每格点），再对 lat/lon 求空间均值（`skipna=True`，无面积权重，与 `cal_mean.py` 口径一致）：
   ```
   annual_cell = da.where(land).groupby('time.year').mean('time', skipna=True)   # (year, lat, lon)
   region_year_cf = annual_cell.mean(('lat','lon'), skipna=True)                 # (year,)
   ```
   - 内存策略：china 文件大，改为逐年 `sel(time=str(year))` 切片读取后再求均值，单年内存 ~2 GB。
   - 夜间太阳 CF=0、风速低于切入风速 CF=0 均为有效值，计入时间平均（CF 本就包含这些 0）。
4. 记录每年 (region, year, mean_cf) 以及辅助字段（n 时间步、n 有效陆地格点）。

**全球逐年平均**：对同一 (energy, scenario, year)，取所有可用区域 `mean_cf`，**按各区域陆地格点数 `n_land_cells` 加权**求平均：
```
global_year_cf = Σ_r (mean_cf_r * n_land_cells_r) / Σ_r n_land_cells_r   # 含 china
```
等价于把所有区域的陆地格点汇到一起求平均。记录 `n_regions`（参与平均的区域数）与 `total_land_cells`（总权重）。

## 4. 输出（存储设计）

**格式：CSV（long format）**。结果量极小（≈ 27 区域 × 46 年 × 3 情景 × 2 能源 ≈ 数千行），CSV 人类可读、便于 pandas/绘图，且与 `utils/cal_mean.py` 现有习惯一致；不采用 nc（对小表格偏重）。

**目录结构**：
```
output/annual_mean_cf/{model}/
├── per_country_annual_cf_{model}.csv     # 各国逐年平均
└── global_annual_cf_{model}.csv          # 全球（N 国 + china）逐年平均
```

**per_country_annual_cf_{model}.csv** 列：
| 列 | 含义 |
|---|---|
| `energy` | solar / wind |
| `model` | 气象模式 |
| `scenario` | ssp126/245/585 |
| `region` | 国家名 / china |
| `year` | 年份 |
| `mean_cf` | 该区域该年平均容量因子 |
| `n_timesteps` | 该年时间步数 |
| `n_land_cells` | 参与平均的陆地格点数（land-mask 开时） |

**global_annual_cf_{model}.csv** 列：
| 列 | 含义 |
|---|---|
| `energy` | solar / wind |
| `model` | 气象模式 |
| `scenario` | 情景 |
| `year` | 年份 |
| `mean_cf` | N+1 区域逐年平均按陆地格点数加权的平均 |
| `n_regions` | 参与平均的区域数 |
| `total_land_cells` | 总陆地格点数（即总权重） |

> 多次运行（不同情景/能源）默认**增量合并**到同一 csv（按 energy+scenario+region+year 去重更新），`--overwrite` 则整表重写。

## 5. 脚本

- 文件：`S06_Annual_Mean_CF.py`（仓库根目录，沿用 S0x 命名）。
- 依赖：`xarray`, `numpy`, `pandas`, `global-land-mask`（均在 `requirements.txt`）。
- 运行示例：
  ```bash
  # 默认：NESM3，三种情景，风+光
  python S06_Annual_Mean_CF.py

  # 只算 ssp245 的光伏
  python S06_Annual_Mean_CF.py --model NESM3 --scenarios ssp245 --energy solar
  ```

## 6. 待定 / 可扩展

- 全球平均为**按陆地格点数加权**。注意该口径隐含假设各区域网格分辨率一致，且未做 cos(lat) 面积权重；如需更严格的"等面积"口径，可后续加 cos(lat) 面积权重。
- 区域内空间平均当前为**无面积权重**（与 `cal_mean.py` 一致）；如需 cos(lat) 面积权重可加 `--area-weight`。
- 若后续重算并掩膜了海洋的 NESM3 文件，`--no-land-mask` + skipna 亦可得到相同结果。
- `_tmp_recompute_global_weighted.py`：临时脚本，可由已算好的 per_country CSV 直接重算加权全球，无需重跑各国计算。

# 附：ERA5-Land 版逐年平均容量因子（S06E02）

## A.1 目标

与上文（`S06E01`，基于 BCSD 降尺度的未来情景）口径完全对齐，但输入换成
**ERA5-Land 再分析**算得的历史 CF，给出 **各国 & 全球的逐年平均容量因子**。
用于把历史（再分析）与未来（降尺度）放在同一坐标系下对比。

- 无 `model` / `scenario` 维度（再分析只有一套数据），用常量列 `source=ERA5Land` 标识。
- 风电、光伏分别独立计算。

## A.2 输入

### A.2.1 ERA5-Land CF 文件（已存在）

```
{cf_root}/CFs_of_{energy}_ERA5Land/{energy}_cf_{year}_{month:02d}.nc
```

- 变量：`solar_cf` / `wind_cf`，维度 `(time, lat, lon)`，**逐小时**。
- **全球** 0.1° 规则网格：`lat` 1801 点（90 → -90，步长 -0.1）、`lon` 3600 点（0 → 359.9，步长 0.1，即 `[0,360)`）。
- **按月分文件**，单文件约 19 GB（如 1 月 744 步）。海洋格点为 NaN（ERA5-Land 仅陆地有值）。
- 年份覆盖 **2015–2025**，其中 **2025 缺 12 月**（为部分年，用 `n_timesteps` 标记）。

### A.2.2 区域网格（关键：保证与各国 BCSD 区域一致）

```
data/grid_of_regions/{region}_grid.nc
```

- 每个文件只含 `lat` / `lon` 坐标（无数据变量），定义该区域的目标网格。
- **必须读取它来界定每个国家的空间范围**，从而与 `S06E01` 的各国区域一致、可直接对比。
- 支持两类网格（脚本按 lat/lon 维度自动识别）：
  - **规则 1D `lat`/`lon`**（如 France 98×148、china 348×442）：正交索引截取，子集 `(rlat,rlon)`。
    经度多为 `[0,360)`，部分国家跨 0° 经线（如 France 同时含 ~0–8° 与 ~355–360°）。
  - **曲线 / 旋转极点 2D `lat(rlat,rlon)`/`lon(rlat,rlon)`**（如 `NAM-12_grid.nc`，628×655=411340 点，
    经度 `[-180,180]`）：把 2D 网格**展平为散点**，逐点求其在 ERA5-Land 0.1° 网格上的最近邻，
    用**逐点索引**截取，子集 `(N,)`。
- ⚠️ **重叠与重复计数**：`NAM-12` 是北美域，与 `México`（及美国陆地）在空间上重叠。
  二者若同时计入**全球加权**会重复计数，且 NAM-12 陆地格点数（~17 万）远大于其他区域、会主导权重。
  如需避免，用 `--exclude-regions NAM-12`（或 `México`）排除其一。脚本默认全部计入，由使用者决定口径。

### A.2.3 命令行参数

| 参数 | 说明 | 默认值 |
|---|---|---|
| `--energy` | `solar` / `wind` / `both` | `both` |
| `--cf-root` | ERA5-Land CF 根目录 | `output` |
| `--grid-dir` | 区域网格目录 | `data/grid_of_regions` |
| `--output-dir` | 结果输出根目录 | `output/annual_mean_cf_ERA5Land` |
| `--years` | 年份范围 `Y0-Y1` | `2015-2025` |
| `--regions` | 仅算指定区域（逗号分隔），空=全部 | 空 |
| `--exclude-regions` | 排除区域（逗号分隔），如 `NAM-12`/`México` 以避免全球加权重复计数 | 空（默认全部计入） |
| `--land-mask / --no-land-mask` | 是否用 global_land_mask 把海洋置 NaN | `--land-mask`（开） |
| `--overwrite` | 整表重写而非增量合并 | 关 |

## A.3 计算方法

对每个 (energy, region, year)，采用 **逐月流式累加**（避免一次载入 19 GB 全球场）：

1. 从 `data/grid_of_regions/{region}_grid.nc` 取该区域 `lat`/`lon`（经度归一化到 `[0,360)` 以对齐 ERA5-Land）。
2. 对该年 12 个（2025 为 11 个）月文件，逐个 lazy 打开全球场，按区域网格截取：
   - 规则网格：用最近邻**正交索引**截取（France 等 0.1° 为精确命中；china 等较粗网格取最近 ERA5-Land 格点）；
   - 曲线 / 旋转极点 2D 网格（NAM-12）：展平为散点后**逐点最近邻索引**截取。
   两者都保证与该区域 BCSD 网格点对齐、区域一致。
3. 仅载入截取后的小立方 `(time, rlat, rlon)`，累加：
   - `sum_arr (rlat,rlon) += nansum_over_time`
   - `cnt_arr (rlat,rlon) += 有限值计数`
   - `n_timesteps += 该月时间步数`
4. 12 个月累加完毕：
   - `cell_mean = sum_arr / cnt_arr`（`cnt=0` 处为 NaN）；
   - **海洋掩膜（默认开）**：global_land_mask 由区域 lat/lon 构建，海洋置 NaN
     （ERA5-Land 海洋本就是 NaN，此步多为冗余，仅为与 `S06E01` 口径一致）；
   - 对 lat/lon 求空间均值（`nanmean`，无面积权重，与 `cal_mean.py` 一致）得该区域该年 `mean_cf`；
   - 记 `n_land_cells`（有限格点数）。

**全球逐年平均**：与 `S06E01` 一致 —— 对同一 (energy, year)，取各区域 `mean_cf`
**按 `n_land_cells` 加权**求平均（含 china）：
```
global_year_cf = Σ_r (mean_cf_r * n_land_cells_r) / Σ_r n_land_cells_r
```

## A.4 输出

```
output/annual_mean_cf_ERA5Land/
├── per_country_annual_cf_ERA5Land.csv     # 各国逐年平均
└── global_annual_cf_ERA5Land.csv          # 全球（N 国 + china）逐年平均
```

- **per_country** 列：`energy, source(=ERA5Land), region, year, mean_cf, n_timesteps, n_land_cells`。
- **global** 列：`energy, source, year, mean_cf, n_regions, total_land_cells`。
- 去重主键：per_country = `energy+source+region+year`；global = `energy+source+year`。默认增量合并，`--overwrite` 整表重写。

## A.5 脚本

- 文件：`S06E02_Annual_Mean_CF_ERA5Land.py`（仓库根目录）。
- 依赖：`xarray`, `numpy`, `pandas`, `global-land-mask`。
- 运行示例：
  ```bash
  # 默认：风+光，2015-2025
  python S06E02_Annual_Mean_CF_ERA5Land.py

  # 只算光伏、2015-2020
  python S06E02_Annual_Mean_CF_ERA5Land.py --energy solar --years 2015-2020
  ```

## A.6 待定 / 可扩展

- 与 `S06E01` 同：全球为按陆地格点数加权、区域内空间为无面积权重；如需更严格可加 cos(lat) 面积权重。
- ERA5-Land 0.1° 远细于 BCSD 网格，china 等较粗区域网格截取时会对 ERA5-Land **降采样**（取最近格点）。
  若希望统计 ERA5-Land 原生分辨率全境，可改为按区域 lat/lon 的 **包络范围** 切片而非逐点最近。当前选用逐点最近以严格对齐各国区域。
